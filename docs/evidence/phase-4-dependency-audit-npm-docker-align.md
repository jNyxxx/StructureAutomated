## 3. Package-manager alignment strategy

The package state was recreated with the approved Next 15 target:

```text
npm install next@15.5.19 eslint-config-next@15.5.19
```

Then `frontend/package.json` was edited only to add the narrow PostCSS override under Next:

```json
"overrides": {
  "next": {
    "postcss": "8.5.16"
  }
}
```

No broad overrides were added. React, Next itself, and unrelated packages were not overridden.

The lockfile was regenerated using Docker's package-manager environment:

```text
MSYS_NO_PATHCONV=1 docker run --rm -v "$(pwd)/frontend:/app" -w /app node:20-alpine npm install --package-lock-only
```

`MSYS_NO_PATHCONV=1` was required because Git Bash was otherwise rewriting `/app` to a Windows Git path. With path conversion disabled, the Docker container mounted the frontend folder correctly and npm `10.8.2` regenerated the lockfile.
